vue的MVVM模式的理解

vue借用了mvvm的设计实现双向绑定，同时需要借助表单元素



MVVM是一种基于前端开发的架构模式，包含三个模块：数据层modle、视图层view以及viewmodel业务逻辑层。核心功能便是提供View和ViewModle的双向数据绑定，ViewModle可以理解为:for当前view的model。通过ViemModel连接View和Model，简化了界面与业务的依赖，解决了数据频繁更新。当modle变化时，ViewModel会自动更新；而ViewModle变化时，View也会变化。
vue的双向绑定参考了MVVM模式，利用Object.defineProperty()以及发布-订阅者模式实现双向绑定，理解defineProperty的作用与观察者模式的原理，对于vue的双向绑定掌握轻而易举：

（1）vue在初始化data选项时会遍历每个属性，利用Object.defineProperty()将他们转为getter/setter，使vue能过追踪依赖，在属性被访问和修改时通知变化。

（2）同时，Vue的指令编译器Compile对元素节点进行扫描和解析，初始化视图，并根据指令模板订阅Wather更新视图替换数据并将自己添加到Dep,并且绑定对于对应的回调函数【针对指令事件】。

（3）在数据改变时，触发Observer中的setter,调用dep的notify,Dep遍历所有的订阅者Watcher,并调用对应的upate函数，订阅者收到通知对视图进行更新。

vue的虚拟dom理解

前端的任务抽象化主要包含两大工作：维护状态[model]、更新视图[view]。
状态以及视图的更新都需要对dom进行操作；mvvm通过ViewModel将视图和状态进行绑定，避免对dom的直接操作，提高开发效率；然而对dom的操作仍然存在，导致了执行效率的降低。在这种场景下，一种兼顾开发效率与执行效率的方案显得尤为重要：虚拟dom。

了解三角函数

在使用三角函数进行canvas动画的应用之前，复习下高中数学关于三角函数的知识点

  • 一弧度 = 弧长等于半径的弧所对应的圆心角的一弧度
  • 360角度 = 2π （一个圆的弧度）
    • 1弧度 = 180 / π； 1角度 = π / 180
  • 正弦函数：圆上任意一点的y坐标和弧长之间的关联
  • 余弦函数：圆上任意一点的x坐标和弧长之间的关联
  • 三角函数(a是对边，b是领边，c是斜边，θ是角度)
    • sin(θ) = a / c
    • cos(θ) = b / c
    • tan(θ) = a / b
  • 反三角函数
    • arcsin(a/c) = θ
    • arccos(b/c) = θ
    • arctan(a/b) = θ
    • Math.atan2(a,b) = θ

      根据维基百科的解释：已坐标原点为起点，指向（x,y）的射线在坐标平面上与x轴方向之间的角的角度

注意：在Math对象中的三角函数采用的弧度制
